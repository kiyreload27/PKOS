# DATA MODEL REFERENCE

## 1. Entity
- **What it represents:** The core anchor of the system. Represents a person, place, concept, task, or thing. It has no strict schema, only an ID and traits.
- **Where it is created:** Originally emitted as a `PKOSEvent` by the Worker when parsing a capture.
- **Where it is read:** In the Web UI (Timeline, Graph, Daily Briefing).
- **Where it is modified:** Never directly modified. New events append traits or metadata.

## 2. PKOSEvent
- **What it represents:** An immutable record of something that happened in the system (e.g., `CAPTURE_RECEIVED`, `ENTITY_CREATED`).
- **Where it is created:** By the Web API (on capture) and by the Worker (on enrichment).
- **Where it is read:** By the Projection Builders to update derived tables, and by the Timeline UI to show history.
- **Where it is modified:** NEVER. It is strictly append-only.

## 3. Relationship
- **What it represents:** A directed edge between two Entities (e.g., `EntityA -> MENTIONS -> EntityB`).
- **Where it is created:** By the Relationship Engine in the Worker.
- **Where it is read:** By the Daily Briefing, Graph, and Entity Detail pages.
- **Where it is modified:** Only updated or deleted via subsequent event logs if proven false.

## 4. Observation
- **What it represents:** A factual claim, insight, or summary generated by AI about an Entity.
- **Where it is created:** By the Observation Engine in the Worker.
- **Where it is read:** Displayed in the AI Inbox (for approval) or directly on the Entity page.
- **Where it is modified:** Can be accepted or rejected by the user in the AI Inbox.

## 5. Memory Card
- **What it represents:** A highly curated, evidence-backed summary of an entity. It only emerges when enough signals (references, views, connections) accumulate.
- **Where it is created:** As a projection derived from multiple underlying events and observations.
- **Where it is read:** The Daily Briefing and Hybrid Search results.
- **Where it is modified:** Automatically rebuilt as new evidence events occur.

## 6. Context
- **What it represents:** A project, workspace, or bounded scope. Used to filter out noise and focus the Daily Briefing.
- **Where it is created:** By the user manually in the Sidebar (`apps/web`).
- **Where it is read:** By the Status Dock, Sidebar, and Daily Briefing ranking layer.
- **Where it is modified:** Toggled active/inactive in the UI.
